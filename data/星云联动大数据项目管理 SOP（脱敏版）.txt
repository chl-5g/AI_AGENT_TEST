星云联动大数据项目管理 SOP（脱敏版）
1. 总则
本规范适用于公司所有大数据开发及系统集成项目。项目全生命周期分为：立项、执行、交付、运维四个阶段。

2. 立项管理规范
2.1 发起条件： 只有通过预研评审的项目方可发起立项。立项申请需提交《项目建议书》及《初步预算表》。

2.2 审批流程： * 预算 < 50万元：由部门经理及技术总监（CTO）审批。

预算 ≥ 50万元：需提交至总经理办公会，经由“三重一大”决策流程审批。

2.3 编号规则： 立项后自动生成编号，格式为 XY-YYYY-顺序号（如 XY-2026-001）。

3. 项目执行与变更
3.1 周报制度： 每周五 17:00 前，项目经理必须在 Jira 系统中提交本周进度，并同步至钉钉群。未按时提交者，扣除当月项目绩效 5%。

3.2 资源补位申请： 若项目进度滞后超过 10%，项目经理可申请资源补位。

申请路径： 在 OA 系统发起“资源调度申请”。

响应时限： 人力资源部需在 2 个工作日内给出初审意见。

3.3 技术变更： 涉及核心架构改动的变更，必须召开“架构评审委员会（ARB）”会议，且过半数专家投赞成票方可执行。

4. 交付与验收规范
4.1 初验必备文档： 1. 《需求跟踪矩阵》
2. 《系统压力测试报告》
3. 《用户操作手册》
4. 《等保三级测评报告》（仅限政府类项目）

4.2 终验条件： 系统上线试运行满 3 个月，且无 P1 级、P2 级未修复故障。

5. 故障等级定义与响应
P1（紧急）： 系统全面瘫痪或数据泄露。响应时限：15 分钟；修复时限：4 小时。

P2（高）： 核心功能受损，影响 30% 以上用户。响应时限：30 分钟；修复时限：12 小时。

P3（中）： 单个模块异常，有绕过方案。响应时限：4 小时；修复时限：3 个工作日。

💡 为什么这份文档适合你的 Agent 练习？
数值判断： 测试 Agent 能否区分“50万以上”和“50万以下”的审批差异。

条件关联： 测试 Agent 是否知道“政府类项目”比普通项目多一个“等保三级”要求。

时间敏感性： 测试 Agent 能否准确抓取“周五 17:00”和“15 分钟”这类硬性考核指标。
